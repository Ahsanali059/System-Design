Builder Design Pattern

"Complex object ko step-by-step construct karna, bina constructor overloads ke."

-------------------------
Problem (Without Builder)
-------------------------

User user = new User(
    "Ali",
    "Khan",
    "ali@gmail.com",
    null,
    null,
    true,
    false
);

=======
Issues:
=======

Confusing parameters

Constructor explosion

Hard to read & maintain

Optional fields problem


==========================
Solution â†’ Builder Pattern
==========================

Step 1: Create Main Class

public class User {

    private final String firstName;
    private final String lastName;
    private final String email;
    private final String phone;
    private final boolean active;

    private User(Builder builder) {
        this.firstName = builder.firstName;
        this.lastName = builder.lastName;
        this.email = builder.email;
        this.phone = builder.phone;
        this.active = builder.active;
    }

}

Step 2: Create Static Builder Class

    public static class Builder {

        private String firstName;
        private String lastName;
        private String email;
        private String phone;
        private boolean active;

        public Builder firstName(String firstName) {
            this.firstName = firstName;
            return this;
        }

        public Builder lastName(String lastName) {
            this.lastName = lastName;
            return this;
        }

        public Builder email(String email) {
            this.email = email;
            return this;
        }

        public Builder phone(String phone) {
            this.phone = phone;
            return this;
        }

        public Builder active(boolean active) {
            this.active = active;
            return this;
        }

        public User build() {
            return new User(this);
        }
    }
}

Step 3: Use Builder (Client Code)

User user = User.Builder()
    .firstName("Ali")
    .lastName("Khan")
    .email("ali@gmail.com")
    .active(true)
    .build();

Advantages:
Readable
Clean
No constructor confusion

Lombok Way (Very Common in Projects)

@Builder
@Data
public class UserDTO {
    private String name;
    private String email;
    private boolean active;
}
UserDTO user = UserDTO.builder()
    .name("Ali")
    .email("a@gmail.com")
    .active(true)
    .build();
